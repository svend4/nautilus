## 1. КОНЦЕПТУАЛЬНАЯ СТАДИЯ

### 1.1 Идея и назначение

Nautilus Portal — **единая точка входа в экосистему репозиториев svend4**. Проблема, которую он решает: у каждого репозитория (`info1`, `pro2`, `meta`, `data2`, `data7`, `infosystems`, `ai_agents`) свой формат данных, своя онтология и своя точка зрения на одни и те же концепты. Без портала пользователь вынужден знать формат каждого репо и искать вручную в каждом из них.

Портал решает это через **адаптерный паттерн**: любой репозиторий реализует два метода (`fetch()` / `describe()`), и становится частью единого поискового пространства.

### 1.2 Теоретическая основа — Q6-пространство

Все концепты экосистемы размещены в **Q6 — 6-мерном бинарном гиперкубе** {0,1}^6 с 64 вершинами. Каждая вершина соответствует:
- одной из 64 гексаграмм И-Цзин (символическое измерение)
- одному из 4 классов клеточных автоматов Вольфрама (динамическое измерение)
- α-уровню абстракции от −4 (конкретный код) до +4 (чистая онтология)

| CA-класс | Поведение | Примеры концептов |
|----------|-----------|-------------------|
| I | Стационарные паттерны | Базовые алгоритмы, ката |
| II | Периодические циклы | Архетипы, методология |
| III | Хаотические | Конфликт, разрушение |
| IV | Сложные (edge of chaos) | Синтез, LCI, революция |

**Метрика близости** — расстояние Хэмминга: два концепта «рядом», если их Q6-координаты отличаются в минимальном числе битов.

### 1.3 Консенсусная модель

При запросе портал опрашивает **все** адаптеры и вычисляет **консенсус** — процент адаптеров, вернувших реальные (не fallback) записи. Консенсус ≥ 100% означает, что концепт подтверждён во всех репозиториях экосистемы одновременно.

## 2. ТЕХНИЧЕСКАЯ СТАДИЯ

### 2.1 Архитектура

```
nautilus/
├── portal.py              # Оркестратор: query, consensus, SPA-сервер
├── api.py                 # REST API (7 эндпоинтов)
├── nautilus_sdk.py        # Python SDK для клиентов
│
├── adapters/              # 13 адаптеров
│   ├── base.py            # Протокол BaseAdapter + PortalEntry
│   ├── info1.py           # svend4/info1 — 74+ Markdown-документа
│   ├── pro2.py            # svend4/pro2 — Q6-концепты (крупнейший, 347 LOC)
│   ├── meta.py            # svend4/meta — 256 CA-правил + гексаграммы
│   ├── data2.py           # svend4/data2 — 310+ томов ЕТД
│   ├── data7.py           # svend4/data7 — граф знаний
│   ├── infosystems.py     # Домен: архитектура ИС
│   ├── ai_agents.py       # Домен: ИИ-агенты
│   ├── auto.py            # Авто-загрузка из nautilus.json
│   ├── obsidian.py        # Obsidian vault ([[wikilinks]])
│   ├── arxiv.py           # arXiv API (12ч кэш)
│   ├── github_topic.py    # GitHub topic discovery
│   ├── jsonl.py           # JSON Lines стриминг
│   └── cache.py           # Дисковый кэш (TTL = 24ч)
│
├── passports/             # 7 паспортов репозиториев (.md)
├── tests/                 # 4 файла, 60 тестов
└── .github/workflows/     # 4 CI/CD pipeline
```
